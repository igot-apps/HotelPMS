# Project Overview: Hotel PMS Backend

## 🎯 Core Purpose
We are building a **production-ready, multi-tenant Hotel Property Management System (PMS) backend**. It serves as the foundation for a complete hotel management solution that scales from a single property to a multi-property, multi-tenant SaaS platform.

## 🏗️ Technology Stack
- **Runtime:** Node.js (v18+)
- **Language:** TypeScript (Strict mode)
- **Framework:** Express.js
- **Database:** PostgreSQL (Containerized via Docker)
- **ORM:** Prisma
- **Validation:** Zod
- **Authentication:** JWT (Access + Refresh tokens) + bcryptjs
- **Security:** Helmet, CORS
- **Logging:** Morgan
- **Testing:** Jest, Supertest

## 🏛️ Architecture Principles
1. **Module-Based Architecture:** Features are isolated in `src/modules/[feature]/`.
2. **Layered Pattern:** Strict separation of concerns: `Routes` -> `Controllers` -> `Services` -> `Repositories`.
3. **Multi-Tenancy:** Strict data isolation. Every query must be scoped to the authenticated user's `tenantId`.
4. **Security First:** JWT auth, RBAC (Role-Based Access Control), PBAC (Permission-Based), password hashing, and soft deletes.
