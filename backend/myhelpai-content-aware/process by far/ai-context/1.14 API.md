# API Documentation

## 🔐 Authentication Flow
1. Client sends `POST /api/auth/login` (username/password).
2. Server returns Access Token (7d) and Refresh Token (30d).
3. Client sends Access Token in `Authorization: Bearer <token>` header.
4. If expired, client sends Refresh Token to `POST /api/auth/refresh-token`.

## 📡 Existing Endpoints (Completed)

### Auth
- `POST /api/auth/login`
- `POST /api/auth/refresh-token`
- `GET /api/auth/me` (Protected)
- `POST /api/auth/logout` (Protected)

### Tenants
- `POST /api/tenants`
- `GET /api/tenants` (Paginated)
- `GET /api/tenants/:id`
- `GET /api/tenants/:id/stats`
- `PUT /api/tenants/:id`
- `DELETE /api/tenants/:id` (Soft delete)

### Properties
- `POST /api/properties`
- `GET /api/properties` (Paginated, tenant filtered)
- `GET /api/properties/:id`
- `GET /api/properties/tenant/:id`
- `GET /api/properties/:id/stats`
- `PUT /api/properties/:id`
- `DELETE /api/properties/:id` (Soft delete)
