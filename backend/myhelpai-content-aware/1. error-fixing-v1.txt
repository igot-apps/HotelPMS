# Hotel PMS Backend Setup Progress Summary (Updated)

## Project Goal

I am building a **multi-tenant Hotel Property Management System (PMS) backend** using:

* Node.js
* TypeScript
* Express
* PostgreSQL
* Prisma ORM
* Docker
* pgAdmin

The system will eventually support:

* Multiple hotels (multi-tenant architecture)
* Reservations and bookings
* Room management
* Guest management
* Check-in / check-out workflows
* Staff and role management
* Reporting dashboards
* Online booking support

At this stage, the focus is on **backend infrastructure setup and tooling stability** before implementing business logic.

## Backend Structure

I built an Express + TypeScript API with:

* Express server setup
* Helmet security middleware
* CORS enabled
* Morgan logging (dev only)
* dotenv configuration
* Health check endpoint (`/health`)
* Root endpoint (`/`)
* Global error handling middleware

### Server Start Logic

The server uses:

```ts
if (require.main === module) {
  app.listen(PORT, () => {
    console.log(`🚀 Server running on http://localhost:${PORT}`);
    console.log(`📊 Environment: ${process.env.NODE_ENV}`);
  });
}
```

This ensures:

* The server only starts when the file is executed directly
* Prevents auto-start when imported in tests or other modules

## Critical Error Encountered (TypeScript)

When running:

```bash
npm run dev
```

The server crashed with TypeScript error:

### Error Type

```
TS6133: 'req' is declared but its value is never read
```

### Affected files

* `/health` route
* `/` root route
* error middleware (`app.use`)

---

## Fix Applied

TypeScript strict mode required unused parameters to be handled properly.

### Solution

Prefixed unused parameters with `_`:

### Before

```ts
(req: Request, res: Response)
```

### After

```ts
(_req: Request, res: Response)
```

### Error middleware fix:

```ts
(err: any, _req: Request, res: Response, _next: any)
```
